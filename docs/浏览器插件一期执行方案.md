# 浏览器插件一期执行方案

## 文档目的

这份文档是一阶段浏览器插件开发的本地执行基线，用来解决 3 件事：

1. 本地可预览当前开发进度
2. 每个阶段完成前有明确的进度把关标准
3. 后续浏览器插件与桌面端 `Immersive-Input` 汇合时，保留可追溯、可复用的流程文档

## 一期范围

一期只聚焦浏览器插件，不扩散到桌面端完整执行。

一期目标是打通以下最小闭环：

- 插件能够接收来自外部 AI / MCP 的任务
- 插件也能够接收自身内置聊天入口发起的任务
- 两类入口统一进入同一套 `Guide Session Runtime`
- 浏览器页面能够展示第一版沉浸式引导 Overlay
- 协议层为未来与 `Immersive-Input` 桌面端 handoff 做好预留

## 当前已有基线

当前项目并不是从零开始，已经具备以下可复用基础：

- `mcp-chrome` 浏览器执行底座
- `AgentChat` 侧边栏入口
- `popup` 轻入口
- `element-marker` 元素标注与锚点管理
- `web-editor` 页面覆盖与编辑基础设施
- `record-replay-v3` 执行、调度、触发器与运行时思路
- `native-host` 本地桥接能力

## 一期交付项

### D1. 共享协议

- 统一的 `Guide Session` 结构
- 统一的 `Guide Step` 结构
- 统一的入口来源、权限模式、运行状态定义
- 面向未来桌面桥接的共享事件契约

### D2. 浏览器运行时

- 后台 `Guide Runtime` 监听器
- `create / list / get / start / advance / cancel / attach-tab` 全链路
- 会话持久化到扩展存储
- 后台到内容脚本的 Overlay 派发

### D3. Overlay MVP

- 页面内引导 Overlay
- 当前步骤面板
- 目标元素高亮
- 会话状态显示
- `Next / Skip / End` 控制按钮

### D4. 入口统一

- 主流 AI / MCP 路径最终可映射到 Guide Runtime
- 插件内置入口最终可映射到 Guide Runtime
- 当前提供 popup 本地预览入口
- 后续所有入口都收敛到同一份任务协议

### D5. Immersive-Input 预留

- Session 结构中保留桌面端桥接字段
- 浏览器侧运行事件可给桌面端复用
- 存在浏览器到 `Immersive-Input` 的共享桥接契约
- 存在本地桥接事件历史，支持追溯与后续回放
- 存在 handoff / accept / resume 生命周期动作
- 不在跨端协议里写死浏览器专属状态

## 一期完成门槛

### 协议门槛

- 共享协议已从 `packages/shared` 导出
- 扩展侧通过共享协议引用 Guide 类型，而不是重复定义
- Session 至少包含来源、目标、权限、状态、步骤、元数据、集成提示

### 运行时门槛

- 后台能创建并保存 Guide Session
- 能查询本地 Session 状态
- 能启动、推进、取消、绑定页面
- 能驱动页面 Overlay 更新

### 页面门槛

- Overlay 能展示当前会话状态
- Overlay 能根据选择器高亮目标元素
- 会话结束时 Overlay 能干净关闭

### 入口门槛

- 至少一条外部入口能发起 Guide Session
- 至少一条内置入口能发起 Guide Session
- 两类入口最终都汇聚到同一个 Guide Runtime

### 复用门槛

- 协议不绑定某一个模型或某一个 AI 平台
- 协议中已保留 `Immersive-Input` 桥接提示
- 运行时中存在桥接事件历史、handoff 状态和恢复动作

## 模块分层

### 共享层

- `packages/shared/src/guide-types.ts`
- `packages/shared/src/guide-bridge-types.ts`
- 用于定义跨浏览器与桌面端的领域协议

### 浏览器运行时层

- `app/chrome-extension/entrypoints/background/guide-runtime/`
- 负责 Session 存储、状态迁移、Overlay 派发、桥接事件落盘

### 浏览器 UI 层

- `app/chrome-extension/entrypoints/guide-overlay.content.ts`
- `app/chrome-extension/shared/guide-overlay/`
- 负责页面展示、交互和局部预览

### 未来集成层

- 内置聊天入口适配层
- MCP 外部任务适配层
- `Immersive-Input` 桥接适配层

## 当前阶段的收口顺序

1. [已完成] 外部 MCP 入口收敛到同一套 Guide Runtime
2. [已完成] 浏览器插件与未来 `Immersive-Input` 共用真实 bridge adapter
3. [已完成] 统一内外入口的任务载荷格式
4. [已完成] 清理历史测试 / 类型债，形成可交接的一期版本

## 一期收口结果

- 一期目标范围已完成，浏览器插件已具备统一入口、统一运行时与页面引导 Overlay 的最小闭环。
- 当前代码库已通过 `pnpm --filter chrome-mcp-server compile` 与 `pnpm --filter chrome-mcp-server build`。
- 一期交付以“功能收口、运行时统一、桥接预留、可交接”为目标，不再继续在一期内增加新功能点。
- 未纳入一期的迁移策略、细粒度错误分类与云端能力，统一转入二期承接。
